    The PKString

    Project maintained by Nicholas Corgan Hosted on GitHub Pages — Theme by mattgraham

    The PKString class is a custom string class used throughout LibPKMN. Using Boost.Locale, it removes the distinction C++ makes between multi-byte strings and wide strings by allowing implicit conversions back and forth with both types. In most situations, the user does not even need to acknowledge the type itself. For example, the factory function for the Item is defined as:

    static sptr make(const pkmn::pkstring &name, const pkmn::pkstring &game);

    But the user can easily instantiate an Item with the following code, which uses C-strings:

    pkmn::item::sptr foo = pkmn::item::make("Great Ball", "Ruby");

    All strings returned from the database by functions in the pkmn::database namespace return their values as PKStrings to preserve any potential Unicode, such as the gender symbols in the Nidorans' species names.

    SWIG

    LibPKMN's SWIG bindings allow the PKString to be used just as conveniently in each language as their own native string classes. The methods of doing so for each language are explained below:

    • Python: SWIG is told to ignore certain functions and rename others to make the class's API as close as possible to that of Python's native string class. SWIG typemaps allow for implicit conversions between these classes.
    • C#: SWIG maps the PKString directly to C#'s native string class, allowing C# users to ignore the class altogether.
    • Java: SWIG maps the PKString directly to Java's native string class, allowing Java users to ignore the class altogether.
